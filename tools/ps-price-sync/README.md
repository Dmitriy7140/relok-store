# PS Store → Google Sheets: синхронизатор цен

Инструмент переносит **все игры из вашего магазина** в Google Sheets и для
каждой автоматически подтягивает актуальную цену в **турецких лирах (TRY)**
с **официального** PlayStation Store (`store.playstation.com/en-tr`).
Сторонние сайты не используются.

## Что делает

- берёт список игр из базы магазина (`server/data/logovo.sqlite`, только `type='game'`);
- для каждой ищет товар в официальном PS Store Turkey и подбирает **нужное издание**
  (Standard / Deluxe / Ultimate / GOTY …);
- сохраняет цену **ровно в формате сайта** — `2.999,00 TL`;
- цену в рублях берёт из магазина и **не меняет**;
- пишет всё в Google Sheets;
- при повторном запуске **обновляет только TRY-цены**, рубли не трогает,
  новые игры добавляет, дубликаты не создаёт;
- если игра не найдена — ставит статус **«Требуется проверка»**,
  если нет цены/товар недоступен — **«Цена недоступна»** (пустых ячеек не бывает).

## Структура таблицы

| ID | Название игры | Издание | Название в PS Store | Цена в турецких лирах (TRY) | Цена в рублях (мой магазин) | Статус | Ссылка PS Store | Обновлено |
|----|---------------|---------|---------------------|-----------------------------|-----------------------------|--------|-----------------|-----------|

Столбец **ID** — служебный ключ (id товара из магазина), по нему идёт upsert
(обновление вместо дублирования). Минимально требуемые столбцы —
«Название игры», «Цена в TRY», «Цена в рублях» — присутствуют.

## Модули

| Файл | Ответственность |
|------|-----------------|
| `storeGames.js` | получение списка игр из магазина (SQLite) |
| `psStore.js`    | поиск игры в официальном PS Store + извлечение цены |
| `matcher.js`    | подбор нужного издания среди кандидатов |
| `priceFormat.js`| парсинг/форматирование турецких цен |
| `sheets.js`     | работа с Google Sheets (создание, upsert) |
| `logger.js`     | логирование (консоль + файл `logs/`) |
| `config.js`     | конфигурация из переменных окружения |
| `sync.js`       | главный сценарий, связывает всё вместе |

## Как работает получение цен (без браузера)

Страница поиска официального стора
`https://store.playstation.com/en-tr/search/<термин>` отдаётся сервером уже
со встроенным состоянием Apollo (`window.__APOLLO_STATE__`), где лежат
объекты товаров с полем `price.basePrice` в виде `2.999,00 TL`. Это надёжнее
приватного GraphQL (меняются хэши) и не требует Playwright.

> **Fallback.** Если Sony уберёт server-side-render, включите Playwright:
> установите `npm i playwright && npx playwright install chromium`,
> и в `psStore.js` замените `fetchWithRetry` на загрузку страницы через
> headless-браузер с ожиданием `networkidle`, затем читайте
> `window.__APOLLO_STATE__` через `page.evaluate`. Остальная логика не меняется.

## Настройка

### 1. Установить зависимости

```bash
cd tools/ps-price-sync
npm install          # ставит googleapis
```

### 2. Создать сервисный аккаунт Google

1. [console.cloud.google.com](https://console.cloud.google.com) → создайте проект.
2. Включите **Google Sheets API** и **Google Drive API**.
3. *IAM и администрирование → Сервисные аккаунты* → создайте аккаунт →
   *Ключи → Добавить ключ → JSON*. Скачанный файл положите как
   `tools/ps-price-sync/google-credentials.json`
   (он уже в `.gitignore`).

### 3. Заполнить `.env` в корне проекta

Скопируйте из `.env.example` блок «Синхронизатор цен» и укажите:

```
GOOGLE_APPLICATION_CREDENTIALS=tools/ps-price-sync/google-credentials.json
GOOGLE_SHARE_EMAIL=ваш-личный@gmail.com      # чтобы видеть таблицу у себя
# GOOGLE_SHEET_ID=...   # оставьте пустым — таблица создастся сама
```

Если у вас **уже есть** таблица — впишите её `GOOGLE_SHEET_ID` и заранее
дайте сервисному аккаунту (его e-mail из JSON, поле `client_email`) доступ
на редактирование.

## Запуск

```bash
cd tools/ps-price-sync
npm run sync
```

или из корня проекта:

```bash
node --env-file=.env tools/ps-price-sync/sync.js
```

При первом запуске:
- создаётся таблица (её ID кэшируется в `.sheet-id`), ссылка печатается в лог;
- заполняются все игры с ценами.

**Повторный запуск** просто снова выполните `npm run sync` — цены в лирах
обновятся, рубли останутся как были, новые игры добавятся.

## Требования

- **Node.js ≥ 22.5** (проект использует встроенный `node:sqlite`).
- Доступ в интернет к `store.playstation.com` и `googleapis.com`.

## Логи

Пишутся в консоль и в `tools/ps-price-sync/logs/sync-YYYY-MM-DD.log`.
Уровень настраивается `LOG_LEVEL=debug|info|warn|error`.
