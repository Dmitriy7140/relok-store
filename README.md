# Релок — магазин PlayStation Турция

Полноценный интернет-магазин для Telegram Mini App: витрина, каталог с поиском
и фильтрами, корзина, избранное и **админ-панель** с управлением товарами,
категориями, подписками, кодами пополнения и медиа.

Проект собран как **full-stack приложение без единой npm-зависимости** —
бэкенд работает на встроенном в Node.js модуле `node:sqlite` и `node:http`.

---

## Быстрый старт

Нужен **Node.js 22.5+** (в нём есть встроенный SQLite).

```bash
cd relok-store
npm start          # или: node server/server.js
```

После запуска откройте:

| Что | Адрес |
|-----|-------|
| Витрина | http://localhost:3000/ |
| Админ-панель | http://localhost:3000/admin |

База данных создаётся автоматически при первом запуске и наполняется
демо-товарами (`server/data/relok.sqlite`).

**Токен админ-панели по умолчанию:** `relok-admin`
Меняется переменной окружения:

```bash
ADMIN_TOKEN="ваш-секрет" PORT=8080 node server/server.js
```

### Демо без сервера

Файл `public/index.html` можно открыть напрямую (двойным щелчком). Без сервера
витрина работает в демо-режиме на встроенных данных (`public/js/seed.js`) —
можно листать каталог, добавлять в корзину и избранное. Для админ-панели и
сохранения данных нужен запущенный сервер.

## Что реализовано

### Витрина (`public/index.html`)
- **Hero-карусель** с автопрокруткой, эффектами scale / fade / parallax и 3D-поворотом, свайпом и точками-индикаторами.
- Разделы на главной: Популярные, Новинки, Акции, **Подписки**, **Коды пополнения**, Игры (раздел «Игровой донат» удалён).
- **Отдельные страницы-каталоги** для каждого раздела с поиском, фильтрами (чипсы), сортировкой (по цене, популярности, новизне, названию) и **пагинацией**.
- Карточки товаров: фото/эмодзи, название, описание, цена и старая цена, скидка, статус наличия, кнопка покупки, hover-эффект.
- Модальное окно товара: для подписок — выбор периода и список преимуществ, для игр — характеристики, для кодов — номинал.
- Корзина, избранное, профиль, оформление заказа (отправка данных боту через `Telegram.WebApp.sendData`).
- Тёмная и светлая темы, skeleton-загрузчики, пустые состояния, тосты-уведомления.
- Адаптив: Mobile → Tablet → Desktop (сетка 2→5 колонок).

### Админ-панель (`public/admin.html`)
- Вход по токену.
- **Товары / Подписки / Коды** — добавление, редактирование, удаление, скрытие без удаления, **drag-and-drop порядок**.
- **Категории** — создание, редактирование, удаление, скрытие, изменение порядка перетаскиванием.
- Редактор подписок с тарифами по периодам и списком преимуществ; редактор игр с характеристиками.
- **Медиа** — загрузка изображений с **автоматическим сжатием** (canvas, до 900px / JPEG), drag-and-drop, удаление.
- Валидация форм, обработка ошибок API, уведомления об успехе.
- Адаптивная вёрстка и переключение темы.

## REST API

Базовый префикс `/api`. Операции записи требуют заголовок `X-Admin-Token`.

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | `/api/products` | Список товаров. Параметры: `type`, `category`, `flag` (`new`/`sale`/`preorder`/`featured`/`instock`), `q`, `sort` (`price_asc`/`price_desc`/`popular`/`new`/`name`/`position`), `page`, `limit` |
| GET | `/api/products/:id` | Один товар |
| POST | `/api/products` | Создать |
| PUT/PATCH | `/api/products/:id` | Обновить (PATCH — частично) |
| DELETE | `/api/products/:id` | Удалить |
| POST | `/api/products/reorder` | Порядок: `{ "ids": [...] }` |
| GET/POST | `/api/categories` | Категории |
| PUT/DELETE | `/api/categories/:id` | Изменить / удалить |
| POST | `/api/categories/reorder` | Порядок категорий |
| POST | `/api/media` | Загрузка `{ data, mime, filename }` (base64) |
| GET/DELETE | `/api/media/:id` | Отдать / удалить изображение |
| GET/PUT | `/api/settings` | Настройки витрины |
| POST | `/api/auth` | Проверка токена админа |

Публичные `GET`-запросы не возвращают скрытые товары и категории; с токеном
администратора возвращаются все.

## Структура

```
relok-store/
├── package.json
├── server/
│   ├── server.js        # HTTP-сервер + REST API + раздача статики
│   ├── db.js            # схема SQLite, помощники запросов, демо-данные
│   └── data/            # файл базы (создаётся автоматически)
└── public/
    ├── index.html       # витрина
    ├── admin.html       # админ-панель
    └── js/
        ├── seed.js      # данные для демо-режима без сервера
        ├── api.js       # клиент REST API с офлайн-фолбэком
        ├── app.js       # логика витрины (роутинг, карусель, каталог)
        └── admin.js     # логика админ-панели (CRUD, drag-and-drop, медиа)
```

## Развёртывание

Сервис — обычное Node-приложение, которое слушает `PORT` и само отдаёт
фронтенд. Подходит для любого хостинга Node (VPS, Railway, Render и т. п.).
Для Telegram Mini App укажите в настройках бота URL вашего домена с
`index.html`. Не забудьте задать собственный `ADMIN_TOKEN`.
